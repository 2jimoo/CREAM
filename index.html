CREAM: Continual Retrieval on Dynamic Streaming Corpora with Adaptive Soft Memory

HuiJeong Son*, Hyeongu Kang, Sunho Kim, Subeen Ho, SeongKu Kang, Dongha Lee, Susik Yoon
Korea University & Yonsei University
KDD 2026 (Jeju Island, Republic of Korea)
Paper (PDF) DOI Code arXiv Resource

Soft memory with query, pseudo-positive samples, and negative samples accross sessions

Abstract

Information retrieval (IR) in dynamic data streams is a crucial task, as shifts in data distribution degrade the performance of AI-powered IR systems. To mitigate this issue, memory-based continual learning has been widely adopted for IR. However, existing methods rely on a fixed set of queries with ground-truth documents, which limits generalization to unseen data, making them impractical for real-world applications. To enable more effective learning with unseen topics of a new corpus without ground-truth labels, we propose CREAM, a self-supervised framework for memory-based continual retrieval. CREAM captures the evolving semantics of streaming queries and documents into dynamically structured soft memory and leverages it to adapt to both seen and unseen topics in an unsupervised setting. We realize this through three key techniques: fine-grained similarity estimation, regularized cluster prototyping, and stratified coreset sampling. Experiments on two benchmark datasets demonstrate that CREAM exhibits superior adaptability and retrieval accuracy, outperforming the strongest method in a label-free setting by 27.79% in Success@5 and 44.5% in Recall@10 on average, and achieving performance comparable to or even exceeding that of supervised methods.

Motivation

In continual retrieval over dynamic streaming corpora, topics and semantics evolve over time. Existing memory-based methods typically rely on a fixed set of predefined queries with ground-truth documents (hard memory), which limits adaptation to newly emerging topics and makes timely human supervision impractical. CREAM introduces adaptive soft memory, which continuously groups and updates semantically related queries and documents, enabling self-supervised learning and robust adaptation to both seen and unseen topics.

Framework Overview

CREAM operates in sessions and repeats three stages: (1) Retrieval using fine-grained token-level similarity (late-interaction style), (2) Memory Update via streaming clustering with regularized prototypes (RP-LSH-based fixed-length prototype), and (3) Encoder Update through self-supervised contrastive learning using pseudo positive/negative samples selected from soft memory with stratified coreset sampling.

  • Fine-grained similarity: token-level interaction improves robustness without external labels.
  • Regularized cluster prototyping: fixed-length prototypes reduce token-wise computation while preserving semantics.
  • Stratified coreset sampling: selects diverse, representative training pairs to generalize across evolving topics.

Experimental Results

We evaluate CREAM on two continual retrieval benchmarks (LoTTE and MSMARCO) under shared-pool evaluation. CREAM consistently outperforms strong unsupervised baselines across sessions and achieves performance comparable to (or even exceeding) supervised continual learning methods. Removing fine-grained similarity and prototype regularization causes the largest drop, indicating that token-level semantics and efficient prototyping are critical in the unsupervised setting. (e.g., LoTTE Success@5 drops from 48.60 to 27.23; MSMARCO Success@5 drops from 73.68 to 44.30).
